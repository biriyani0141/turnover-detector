# Decision Log — turnover-detector

技術的・設計的な重要判断を時系列で記録する。
ルール: **追記のみ。過去エントリは編集・削除しない。**
判断が覆った場合は新エントリで上書きを宣言する。

書式: 日付 / 決定 / 理由 / 影響

-----

## YYYY-MM-DD 時価総額は raw close × shares で計算する

- **決定**: AdjCを使わず、生の終値 × 発行済株式数で算出。
- **理由**: AdjCは過去遡及で変動し、時価総額フィルタの閾値がブレて
  再現性が失われるため。
- **影響**: 時価総額フィルタ全般。

## YYYY-MM-DD リターン計算は AdjC を使用する

- **決定**: 騰落率・パフォーマンス系はAdjCを使う。
- **理由**: 分割・配当の影響を除かないと連続したリターンにならない。
- **影響**: 5期間パフォーマンスブロック、各種スクリーニング。

## YYYY-MM-DD 銘柄コード結合は文字列完全一致のみ

- **決定**: コード結合時に変換を一切せず、文字列完全一致でjoin。
- **理由**: 英数字混在コードで実データ汚染事故が発生したため。
  変換は事故の温床。
- **影響**: 全データ結合処理。J-Quants APIからの全取り込み経路。

## YYYY-MM-DD Claude Code は無指示の git 操作を実行しない

- **決定**: git操作はユーザーの明示指示時のみ。自律実行を禁止。
- **理由**: 意図しない履歴汚染・誤pushを防ぐ。GitHub操作は手動運用。
- **影響**: Claude Code CLIの全作業フロー。

## YYYY-MM-DD UIカラーは日本市場慣習に従う

- **決定**: 上昇=赤 / 下落=緑 / 異常急騰=オレンジ / ダーク基調。
- **理由**: 国内トレーダーの直感に合わせる。欧米慣習（緑=上げ）の逆で
  実装ミスが起きやすいため明文化。
- **影響**: 全UI、StockRow.tsx、各種数値表示。

## YYYY-MM-DD 出力JSONは3ファイル構成

- **決定**: ranking.json / popular.json / appearance.json に分割。
  240日・約1,500銘柄・窓は 25/50/100/200営業日。
- **理由**: 画面（回転率ランキング / 人気継続）と用途ごとに責務を
  分離した方がフロントの読み込みと責務が明確。
- **影響**: バックエンド出力層、フロントのデータ取得層。

## YYYY-MM-DD UIプロトタイプ v2 を実装の基準とする

- **決定**: prototype-v2.html（Tailwind/CSS実寸指定済み）を確定版とし、
  新規UI実装はこれを正として差分で行う。
- **理由**: デザインを二重管理しないため。HTMLが実質の仕様書として機能。
- **影響**: 全フロント実装、別Claudeインスタンスへの引き継ぎ。

## 2026-06-22 /pullback の状態分類に「中立帯」を追加

- **決定**: `/pullback` の `classify()` に新状態「中立帯」を追加。
  判定条件は `s1y === "+" && s3m === "0"`。
- **理由**: s1y="+" を前提とした s3m の3分岐（+/0/-）のうち、
  `s3m === "0"` のケースがどの既存条件にも一致せず対象外に
  フォールスルーしていた空隙を切り出したもの。判定軸は「失速」
  （`s1y === "+" && s3m === "-"`）との対称性に合わせ s3m 単軸とし、
  s1m・s5d までは見ない。
- **影響**: `/pullback` の状態セクション構成。表示順は
  調整予備軍 → 中立帯 → 失速 の間に配置、帯色はグレー
  （`bg-gray-700`）。なお s1y が "0"／"-" のケースは中立帯に
  含めず対象外のまま（母集団の意味を「長期上昇中の押し目候補」に
  保つため、長期下落・長期中立の銘柄は混ぜない）。
